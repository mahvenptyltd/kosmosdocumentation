km_add_path_parameters(pstr_url, pstr_path_parameters, bln_encode=False, **kwargs)¶
Description:
This method adds Path Parameters to the base URL and returns the final URL
Parameters

  • pstr_url (String) – url for which path parameters need to be added

  • pstr_path_parameters (String) – parameters that need to be added

  • bln_encode (bool) – False will add the parameters as it is True replaces special characters in the url using %xx escape. Letters, digits and characters ‘_._’ are never quoted

  • kwargs (**kwargs) – **kwargs takes one key to specify if you want to replace or append a parameter Accepted value: replace_Params - False by default. True if you want to replace a parameter

Returns

str_url_with_params - string URL with parameters

Examples:
1. add_path_parameters(“https://www.samplesite.com”, “param1/param2”) returns https://www.samplesite.com/param1/param2
2. add_path_parameters(“https://www.samplesite.com/param1/param2”, “param3”) returns https://www.samplesite.com/param1/param2/param3
3. add_path_parameters(“https://www.samplesite.com/param1/param2”, “param3”, pbln_replace_params=True) returns https://www.samplesite.com/param3

Note

Default value of **kwargs parameter replace_Params is False

km_generate_headers_from_string(pstr_header_string, **kwargs)¶
Description:
This Method converts the headers string to a dictionary.
you need to send header key and values with separators and mention those separators in the **kwargs parameters as shown in example
Parameters

  • pstr_header_string (String) – header string

  • kwargs (**kwargs) – kwargs to send one or more delimiters

Returns

dict_headers

Examples:
1. generate_headers_from_string(“name:abc,age:50”) - No Kwargs hence default delimiters will be used.
2. generate_headers_from_string(“name:abc#age:50”, pstr_headers_delimiter=”#”) - default delimiter for values will be used.
3. generate_headers_from_string(“name#abc,age#50”, pstr_values_delimiter=”#”) - default delimiter for headers will be used.
4. generate_headers_from_string(“name#abc;age#50”, pstr_headers_delimiter=”;”, pstr_values_delimiter=”#”) - delimiters from parameters will be used.

Note

**kwargs accepts the below values as its keys.
1. Key : pstr_headers_delimiter, default value : comma (,)
2. Key : pstr_values_delimiter, default value : colon (:)

km_set_client_settings(pbln_strict=True, pbln_raw_response=True, pbln_force_https=True, pdict_extra_http_headers=None, pbln_xml_huge_tree=False, pbln_forbid_dtd=False, pbln_forbid_entities=True, pbln_forbid_external=True, pbln_xsd_ignore_sequence_order=False)¶
Description:
Before a soap service is called, you will be using an object called client object
This object will have its configuration settings which helps you set various options
Parameters

  • pbln_strict (bool) – boolean to indicate if the lxml should be parsed a ‘strict’. If false then the recover mode is enabled which tries to parse invalid XML as best as it can.

  • pbln_raw_response (bool) – boolean to skip the parsing of the XML response by zeep but instead returning the raw data

  • pbln_force_https (bool) – Force all connections to HTTPS if the WSDL is also loaded from an HTTPS endpoint. (default: true)

  • pdict_extra_http_headers (dictionary) – dditional HTTP headers to be sent to the transport. This can be used in combination with the context manager approach to add http headers for specific calls.

  • pbln_xml_huge_tree (bool) – disable lxml/libxml2 security restrictions and support very deep trees and very long text content

  • pbln_forbid_dtd (bool) – disallow XML with a <!DOCTYPE> processing instruction

  • pbln_forbid_entities (bool) – disallow XML with <!ENTITY> declarations inside the DTD

  • pbln_forbid_external (bool) – disallow any access to remote or local resources in external entities or DTD and raising an ExternalReferenceForbidden exception when a DTD or entity references an external resource.

  • pbln_xsd_ignore_sequence_order (bool) – boolean to indicate whether to enforce sequence order when parsing complex types. This is a workaround for servers that don’t respect sequence order.

Returns

returns Setting object

Note

Default values for the parameters are: pbln_strict=True, pbln_raw_response=True, pbln_force_https=True, pdict_extra_http_headers=None, pbln_xml_huge_tree=False, pbln_forbid_dtd=False, pbln_forbid_entities=True, pbln_forbid_external=True, pbln_xsd_ignore_sequence_order=False
pbln_raw_response=True will fetch the response object with status code, response, url, method
