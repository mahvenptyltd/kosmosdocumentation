km_mouse_operations(element, pstr_mouse_action_type, **kwargs)¶
Description:
This method will be used to perform the mouse operations (left click, right click, double click, move, drag)
In this method, user will provide the object of the element.
By using that object, another method find_central_position (which is written to calculate the central position of element) will return the coordinates and then perform the specified operation.
For Drag, user can specified the object of desitination element or can provide the coordinates of the destination element to perform the operation
Parameters

  • element (Object) – This parameter will have a object which has details of the specified element

  • pstr_mouse_action_type (String) – This parameter will have the mouse action type in string value e.g. left,right,double

  • obj_destination (Object) – This parameter will have the object of element has details of the specified element

  • pstr_coordinates (String) – This parameter will have the values of coordinate[x,y] in the string format. Comma(,) will be used to separate the x and y values

Examples:
pstr_window_title=’KeyDev Collection’
app = Windowsfuntions.km_connect_to_app(pstr_title_re=pstr_window_title)
main_window = app.top_window()
element = Windowsfuntions.km_get_element(main_window,’control_type’,’Edit’,pstr_auto_id=’txtBoxEventType’)
class_obj.km_mouse_operations(element,’click’,pstr_click_type=’double’)

class_obj.km_mouse_operations(element,’click’,pstr_click_type=’left’)

class_obj.km_mouse_operations(element,’drag’,pstr_coordinates=’181,694’)
class_obj.km_mouse_operations(element,’move’)

obj.km_mouse_operations(element,’drag’,obj_destination=element2)

Note

If user will not specify any click type, this method will consider the default value which is ‘left’
And it will perform the left click operation

km_scrolling(**kwargs)¶
Description:
This method is used for basic scroll functionality where the user provides the element/section where scroll bars exist,
along with scroll count and direction. And on the basis of provided parameters, the scroll will perform
For custom controls where user can face issues in unique identification of element/section, he/she can use parent_element parameter in this case.
Parameters

  • pstr_scroll_direction (String) – This parameter will be used to pass the scroll direction e.g. up, down, left or right

  • pstr_scroll_count (Interger) – This parameter will be used to pass the count of scroll, means how many times user wants to scroll down/up/left/right

  • parent_element (Object) – This parameter will be used to pass the object of parent element. It will be used when unique identification of element will be difficult

  • scrollable_element (Object) – This parameter will be used to pass the element e.g. specific section where scroll bar applies

Returns

None

Examples:
obj.km_scrolling(scrollable_element=elemnt,pstr_scroll_count=10,pstr_scroll_direction=’down’)
obj.km_scrolling(scrollable_element=scrollable_element, parent_element=parent_elemnt,pstr_scroll_count=5,pstr_scroll_direction=’up’)

Note

For custom controls, this method will work when only 1 scroll bar exists. This is not applicable on section/window where multiple scroll bar exists
For windows/non-custom Controls: This method will work in both single and multiple scroll bars window/sections
